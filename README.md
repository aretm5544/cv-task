# Dodo Table Cleaning Detection Prototype
Прототип системы компьютерного зрения для автоматического отслеживания статуса столиков в ресторане. Скрипт анализирует видеопоток, фиксирует подходы и уходы гостей, а также вычисляет среднее время простоя столика между посетителями.
---

## Установка и запуск

* Склонируйте репозиторий и перейдите в папку с проектом.

- git clone https://github.com/aretm5544/cv-task

* Установите необходимые зависимости:

- pip install -r requirements.txt

* Запустите основной скрипт, передав путь к видеофайлу:

- python main.py --video <имя_видеофайла>.mp4

(При запуске появится окно выбора зоны — выделите нужный столик мышкой и нажмите ENTER )

## Логика детекции (Design decisions)
* Для обеспечения баланса между скоростью работы и точностью был выбран следующий пайплайн:

* Модель: Используется легкая нейросеть YOLOv8n, которая обеспечивает обработку видео в реальном времени, после неудачных вариантов я заменил модель на более сложную (мега), однако скорость выполнения кратно увеличилась и я принял решение использовать лёгкую модель, но с некторыми доработками 

* ByteTrack (Трекинг): Вместо покадровой детекции используется метод model.track(). Это позволяет присваивать ID объектам и не терять их из виду при кратковременном снижении уверенности модели (confidence).

Адаптивный порог (Confidence Threshold): Порог уверенности снижен до conf=0.15. Это осознанное решение для сложных ракурсов (например, когда гость сидит спиной к камере или его одежда сливается с фоном).

Проверка по центру масс: Вхождение в зону стола (ROI) проверяется не по всей площади bounding box и не по ногам, а по координатам центра рамки (cx, cy). Это дает наибольшую точность для гостей, находящихся в сидячем положении.

Гистерезис (Защита от "дребезга"): Внедрен счетчик CONFIRM_FRAMES. Статус столика ("Empty" / "Occupied") изменяется только в том случае, если система фиксирует новое состояние стабильно на протяжении 4 секунд (после всех экспериментов я пришёл к 4 секундам). Это отсеивает ложные срабатывания от проходящих мимо людей или кратковременных перекрытий (окклюзий).

## Результаты работы

* В процессе выполнения скрипт генерирует видеофайл output.mp4 с визуализацией работы модели (отрисовка bounding box и статуса стола).

* События записываются в Pandas DataFrame и экспортируются в results.csv.

* Среднее время задержки (между уходом гостя и подходом следующего): 212 секунд

## Проблемные кадры

* В ходе работы системы были выявлены факторы, усложняющие детекцию легкими моделями:

* Окклюзия элементами интерьера: На некоторых ракурсах массивные лампы, висящие над столами, перекрывают гостей при вставании, что заставляет модель временно "терять" человека.

* Сложные позы и ракурсы: . На приложенном скриншоте видно, что когда гость сидит спиной к камере в одежде, сливающейся с фоном, видимая площадь (bounding box) сильно сокращается. Это приводит к падению уверенности детекции (confidence) ниже стандартных порогов. Проблема была решена путем снижения порога и использования встроенного трекера. Также во втором видео (с которым производилась основная работа) - я столкнулся с проблемой, что в 10:36 Женщина подкатывает детский самокат к столику и ложит детскую игрушку на стол , то есть занимает столик и сразу после этого уходит делать заказ, но что-то её не устроило и она в конце концов уйдёт из ресторана вовсе, то есть человека за столиком не было и для моего скрпита столик был свободен. (Два примера проблемных кадра представлены в папке проекта "dificult_cards")
